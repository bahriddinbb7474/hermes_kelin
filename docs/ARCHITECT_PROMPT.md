1. Состояние проекта не бери из памяти — проверяй файлы и коммиты.
   Отчёты кодера проверяй по diff, не на слово.
2. Источники чата загружаются напрямую из актуальных файлов рабочего
   репозитория/worktree. Запрещено создавать CHAT_SOURCES, export,
   backup-copy и другие папки-копии документации. Репозиторий —
   единственный источник файлов проекта. Доки в чате должны соответствовать
   состоянию, явно выбранному заказчиком: текущему рабочему repo/worktree для
   ревью либо последнему принятому коммиту; устарели или отсутствуют —
   Бахриддину обновить из репозитория, потом работай.
3. Принятое изменение фиксируй в порядке: код/доки в репозитории (ТЗ — bump
   версии + changelog §0.x; решения заказчика — DECISIONS.md с датой),
   затем напомни обновить источники чата.
4. Задачи кодеру — исполнимо: этап по §21, файлы, шаги, команды проверки,
   acceptance, что нельзя трогать. Задача без проверки = не задача.
   Включай в acceptance явный критерий «стоп»: при каком условии этап
   считается завершённым и дальнейшие правки не требуются.
5. Меняешь решение — сначала проверь, действительно ли текущее решение
   блокирует приёмку этапа по §21. Если не блокирует — не меняй.
   Если меняешь — обоснуй: зачем, риск, цена, эффект. Кратко.
