# Cron And Reminders

Источник истины: `TZ_Hermes_Mariyam_FINAL_v3_0.md`

## Главное правило

Все расписания и напоминания идут только через встроенный Hermes cron. Отдельный backend scheduler не создавать.

## Что планирует Hermes

- Утреннее сообщение: салом, погода, короткий план, мягкая фраза.
- Вечерний вопрос: расходы, Коран, лекарства/самочувствие, если данных не было.
- Ежедневные новости: 3-5 коротких пунктов узбекской кириллицей, без тревоги.
- Админ-отчёт в 19:30 по Asia/Tashkent.
- Onboarding: один шаг в день, с сохранением `onboarding_step`.
- Heartbeat админу: система работает, status ok.
- Пользовательские напоминания: лекарства, Коран, бытовые дела, звонки, намаз, custom.

## Таблица cron-задач для настройки (Этап 6)

Все расписания — в **Asia/Tashkent**. Время утра/вечера/новостей — предложение по умолчанию, согласовать с Бахриддин ака до включения.

| Задача | Время (предлож.) | Получатель | Данные/tools |
|---|---|---|---|
| Утреннее сообщение | 08:30 ежедневно | Ойижон | погода (OpenWeather, кэш), намаз (Aladhan Hanafi, кэш), план дня |
| Onboarding-шаг | вместе с утренним | Ойижон | память: `onboarding_step`, `onboarding_last_shown_date`; пока step < 5 |
| Новости дня | 10:00 ежедневно | Ойижон | веб-поиск Hermes (kun.uz, gazeta.uz, daryo.uz), кэш на день |
| Админ-отчёт | 19:30 ежедневно | Бахриддин ака | `get_admin_report_data` → прозу пишет Hermes |
| Вечерний вопрос | 20:30 ежедневно | Ойижон | только если за день не было данных (расход/Коран/самочувствие); не повторять, если молчит |
| Heartbeat | 09:00 ежедневно | Бахриддин ака | `get_bot_status`; текст: `Мариям тизими ишлаяпти. Ҳаммаси жойида.` |

Пользовательские напоминания создаются из речи в любой момент — отдельной таблицы не требуют.

## Как создаются напоминания

Ойижон или Бахриддин ака говорят естественной фразой. Hermes понимает время и смысл, создаёт cron-задачу и подтверждает простыми словами.

Примеры:

- `Эртага соат 10 да дорини эслат.`
- `Завтра в 10:00 напомни маме выпить лекарство.`

## Лекарства в MVP

Отдельный medication-модуль не делаем: нет назначенного списка. Просьба напомнить лекарство — обычное cron-напоминание.

## Данные и backup

Тексты-заметки и счётчики, если нужны как факт, сохраняются через `save_plan_note`. Сами cron-задачи и напоминания живут в Hermes profile; поэтому backup должен включать копию профиля `mariyam_oyijon`.

## Конкретные промпты для Hermes cron (при деплое)

Все задачи — часовой пояс **Asia/Tashkent**. Времена — по таблице выше (v3.1).
Создаются один раз при setup профиля (ТЗ §21).

### Утро (ежедневно 08:30)
```
Отправь Ойижон утреннее сообщение: салом (Ассалому алайкум, Ойижон),
погода Ташкент (OpenWeather, кэш), намаз (Aladhan Hanafi, кэш),
короткий план дня, мягкая фраза. Если onboarding_step < 5 — вместо
плана отправь текст шага онбординга (skill mariyam §5) и увеличь
onboarding_step. Только узбекская кириллица. Без тревоги.
```

### Новости (ежедневно 10:00)
```
Найди 3–5 свежих новости Узбекистана (kun.uz, gazeta.uz, daryo.uz),
кратко перескажи узбекской кириллицей, без тревожных заголовков.
Кэш на день — не дублируй. Отправлять БЕЗУСЛОВНО каждый день (ТЗ §9.1).
```

### Админ-отчёт (ежедневно 19:30)
```
Бахриддин ака (русским допустимо): расходы за день (через get_admin_report_data),
алерты (medical/urgent), статус системы. Кратко, по фактам. Ежедневно.
```

### Вечер (ежедневно 20:30)
```
Спроси Ойижон мягко: были ли сегодня расходы, Коран, лекарства/самочувствие.
Только если за день не было данных — не повторять, если молчит.
Только узбекская кириллица.
```

### Heartbeat (ежедневно 23:00)
```
Бахриддин ака — БЕЗУСЛОВНО каждый день (ТЗ §18.3):
«Мариям тизими ишлаяпти. Ҳаммаси жойида.»
Если heartbeat НЕ пришёл — бот молча упал, это сигнал тревоги.
Условная отправка запрещена.
```

### Onboarding (часть утра, ежедневно)
```
См. skill mariyam §5: один шаг в день, onboarding_step 0..5.
После шага 5 — не показывать. Устойчивость к паузам.
```
